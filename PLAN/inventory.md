The recommended home-manager approach:

Do NOT use nixvim or manage plugins via Nix. LazyVim self-manages plugins.

### Script notes

The current `flake.nix` has a `wrapScript` helper that:
1. Copies script to `$out/bin/<name>`
2. Makes it executable
3. Wraps it with `wrapProgram` to set PATH to include Ruby + ripgrep + $out/bin

The current derivation only wraps `manners`. Phase 10 extends this to all
Ruby scripts.

Most scripts have hardcoded shebangs pointing to ASDF Ruby:
`#!/Users/millerhall/.asdf/installs/ruby/3.4.3/bin/ruby`

These shebangs are irrelevant when scripts are wrapped via Nix's `wrapProgram`
(the wrapper sets up the environment before invoking the script), but they
should be updated to `#!/usr/bin/env ruby` as a cleanup step for when scripts
are invoked without the Nix wrapper.

`bin/lib/` contains shared Ruby modules: `git.rb`, `git_jj.rb`, `helpers.rb`,
`notes.rb`, `workspace_each.rb`. These must be included in the Nix derivation.
The current flake already does `cp -r ${./bin/lib} $out/bin/lib`.
